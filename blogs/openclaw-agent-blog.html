OpenClaw + WhatsApp on Local Machine

February 14, 2026 8 min read
AI Agents OpenClaw WhatsApp
This post is for anyone who wants to run an AI agent on their local machine and connect it with WhatsApp. If you are a beginner, this guide is made for you.

I recently tried OpenClaw and connected it to WhatsApp. It was simpler than I expected, and the result felt like a practical first step toward a personal Jarvis-style assistant.

In my case, I call my assistant Chanakya. The idea is simple: if we want real control over our own AI systems in the future, we should start understanding these tools now while they are still evolving.

What Is OpenClaw?

OpenClaw is a local-first agent runtime. Think of it as an early, practical version of a personal AI operator that can chat, use tools, and connect to channels like WhatsApp, Telegram, Slack, and more. This is still early-stage tech, but it is already useful.

0) Prerequisites

At the time of writing (February 14, 2026), OpenClaw setup expects Node.js v22+.

  • Use nvm so you can manage multiple Node versions cleanly.
  • Different projects can need different Node versions, so this keeps your system maintainable.

Install nvm

curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash

Reload your shell

# zsh
source ~/.zshrc

# bash
source ~/.bashrc

Install and use Node

nvm install 24.13.1
nvm use 24.13.1
nvm alias default 24.13.1

1) Install OpenClaw

curl -fsSL https://openclaw.ai/install.sh | bash

This script installs OpenClaw and required components. If installation fails with npm/node-related errors, re-check your Node version first.

2) Understand 3 Core Concepts

Gateway Host

The Gateway is the long-running core service where your agent lives. On local setup, this is typically: http://127.0.0.1:18789/

Channel

A channel is a connector between the Gateway and external chat platforms (WhatsApp, Telegram, Slack, and others).

Skill

A skill is like a capability plugin. It teaches your agent how to do useful actions beyond plain chatting, such as calling tools/APIs for specific tasks.

3) Onboard OpenClaw

openclaw onboard --install-daemon

Use --install-daemon if you want OpenClaw running in the background all the time. This means you do not need to manually start it on every terminal session.

During onboarding, choose:

  1. Provider: OpenAI
  2. Auth method: OpenAI Codex (ChatGPT OAuth)
  3. Default model: keep the recommended openai-codex/gpt-5.3-codex
  4. QuickStart channel: WhatsApp (QR link)

4) Link WhatsApp

Scan the QR code from terminal using WhatsApp Linked Devices on your phone. Credentials are stored locally under: ~/.openclaw/credentials/whatsapp/default/

If this is your personal number, selecting personal phone mode sets a stricter default (allowlist behavior) so unknown senders are not automatically accepted.

Do not share tokenized dashboard URLs or personal phone numbers in public screenshots/blogs. Rotate credentials if something sensitive is exposed.

5) If Health Check Fails Once, Don't Panic

In my run, I briefly saw: gateway closed (1006 abnormal closure). After restart/retry, WhatsApp linked and session became usable.

  • Open dashboard and confirm top-right status shows Health OK.
  • Check logs at ~/.openclaw/logs/gateway.log.
  • If needed, run onboarding again and relink channel.

6) First WhatsApp Test

Prompt: "I want to send WhatsApp message"
Then provide: recipient number + message text

Example message input: 

+9194XXXXXXXX
Hello World!

Once linked, you can message your agent remotely through WhatsApp while you are away from your desk.

7) Keep It Running

For continuous availability, keep your machine online. If the host machine is off or sleeping deeply, your agent will not process incoming messages until it is back online.

Final Thoughts

OpenClaw setup is beginner-friendly if you follow steps in order. Start small, keep security settings strict, and gradually add channels/skills. This is an early phase of personal AI infrastructure, so experimenting now gives you a big advantage later.
